# Implementation Prompt

Use this when moving from plan to code.

## Requirements

- Keep edits scoped to the current task.
- Reuse existing shared helpers before inventing a new abstraction.
- If adding or changing a component, check whether a shared UI primitive should own it.
- Update both API and UI when a response contract changes.

## Completion

- Summarize the user-visible change.
- Note any follow-up that should happen in CI, deploy, or environment config.
